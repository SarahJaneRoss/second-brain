Opening Claude Code in Your Project
Terminal Survival Kit

Do this every time you open a new terminal window

1

Open a new terminal window

2

Type: cd ~/claude-code-course/playground/second-brain

3

Type: claude

Why this matters: Claude Code needs to be opened inside your project folder so it knows where all your files are. If you just type claude from anywhere, it won't know about your second brain.

Key / Command What it does
Ctrl + C Cancel whatever is running. Your get-out-of-jail key.
Ctrl + D "I'm done typing." Used after pasting into import_log. Nothing visible happens — it just submits.
source ~/.zshrc Reload your settings after making changes. Only needed once per terminal session.
pwd "Where am I?" — prints your current folder location.
cd ~ Go back to your home folder from anywhere.
history Show a list of everything you've typed recently.
↑ arrow key Bring back your last command so you can re-run or edit it.
Gotchas — Things That Will Trip You Up
!

Apostrophes break commands — Don't use ' in your text. todo chris' notes will hang. Use todo chris notes instead. If it hangs, press Ctrl+C to cancel.

!

The quote> prompt means the terminal is waiting — If you see quote> instead of the normal prompt, you accidentally opened a quote that never closed. Press Ctrl+C to get out.

!

Aliases only work after sourcing — If you edit ~/.zshrc and your new command isn't working, run source ~/.zshrc to reload it. New terminal windows load it automatically.

★

Your journal lives at ~/second_brain/ — Files like journal_2026-03-25.md are plain text files. You can open them in any app including VS Code or Notes.

★

The second brain aliases work from any terminal window — You don't need to be in your project folder to use todo, log, journal, etc. Those work anywhere.

★

You only need Claude Code open for standup — All the capture commands (todo, log, import_log) work without Claude. Open Claude Code at end of day when you're ready to generate the report.

At a Glance

During the day (no Claude needed)

  • todo <item> — capture a to-do as it hits you
  • log <what happened> — log a meeting, task, or note in real time
  • import_log — paste in your CMS work log whenever you have it

End of day (open Claude Code first)

  • Navigate to project: cd ~/claude-code-course/playground/second-brain
  • Open Claude: claude
  • Generate report: standup
  • Copy output → paste into Basecamp check-in

When something goes wrong

  • Terminal frozen or stuck → Ctrl+C
  • Command not found → source ~/.zshrc
  • Apostrophe hung the terminal → Ctrl+C, retype without '
  • Not sure where you are → pwd
