Claude Code — Built-in Slash Commands

A quick reference for the most important / commands in Claude Code.
Visit code.claude.com for full documentation.

Quick Start
Session Management

Getting Started

Type / at the Claude Code prompt to see all available commands.

Type / followed by any letters to filter the list.

Commands work in any interactive Claude Code session — terminal, IDE, or desktop app.

/clearClear conversation history and free up context | /reset, /new
/contextVisualize current context usage as a colored grid with optimization tips
/copyCopy last response to clipboard; interactive picker for code blocks
/desktopContinue session in Claude Desktop app | /app
/diffInteractive diff viewer for uncommitted changes and per-turn diffs
/effortSet effort level: low medium high max auto
/exitExit the CLI | /quit
/exportExport conversation as plain text to a file or the clipboard
/forkCreate a fork of the current conversation at this point
/modelSelect or change the AI model
/remote-controlMake session available for remote control from claude.ai | /rc
/rewindRewind conversation and/or code to a previous point | /checkpoint

Beyond Built-in Commands

Bundled Skills

/simplify, /batch, and /debug ship with Claude Code and appear alongside built-in commands in the / menu.

Custom Skills

Any .md files placed in .claude/skills/ or installed via plugins become /skill-name commands you can invoke.

MCP Prompts

MCP servers can expose prompts as /mcp__<server>__<prompt> commands, dynamically discovered from connected servers.

Power Tips
1

Save context with /clear — Long sessions eat context fast. Clear history when switching tasks to reclaim space and keep Claude focused.

2

Use /fork before experimenting — About to try something risky? Fork first. You can always go back to the fork point if things go sideways.

3

Ask side questions with /btw — Need a quick answer without derailing your conversation? /btw keeps side questions separate from main context.

4

Tune performance with /effort — Use low for quick factual lookups, high for complex reasoning, and max (Opus only) for the hardest problems.

5

Monitor usage with /context — The colored grid shows exactly how full your context window is. Use it to decide when to /clear or /export.

6

Try commands in a separate session — Open a second Claude Code session to experiment with unfamiliar commands. Your main session stays untouched while you explore.
